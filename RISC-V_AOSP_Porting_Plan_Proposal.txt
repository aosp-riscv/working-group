Here is a draft proposal for RISC-V AOSP Android Porting Plan.
"Core Porting work" will focus on bringing up a minimum android rootfs with RISC-V Kernel image. It should be able to start console and native binary. And it will provide a solid build environment and runtime environment for further developing work.
"Base Porting work" will focus on bringing up Android Java Runtime with APK support.
"Extened Porting work" will focus on making up Android framework service into full function.

1. Core Porting work items

1.1 RISC-V AOSP Android GCC tool chain
Upgrade the gcc toolchain with RISC-V Android GCC version.
AOSP reference as https://android.googlesource.com/toolchain/gcc/
$repo init -u https://android.googlesource.com/platform/manifest -b gcc
$repo init -u persistent-https://android.git.corp.google.com/platform/manifest -b gcc
